D4rK API Console code of conduct

This workspace exists to maintain high-quality JSON APIs for D4rK's Android applications. Follow these guidelines to keep collaboration respectful and to ensure releases remain stable.

Working together

  • Treat contributors with respect. Feedback should be constructive and focus on improving the API output, not the person.
  • Use inclusive language across comments, issues, and commit messages.
  • Avoid sharing sensitive data in sample payloads. Use placeholder values when demonstrating schemas.

Change review principles

  • Validate generated JSON through the provided builders before committing files to main.
  • Reference documentation links in the PR description when changing schemas or introducing new content types.
  • Keep history clear by using descriptive commit messages and avoiding force-pushes on shared branches.

Escalation

Report unacceptable behaviour or policy concerns to Mihai-Cristian Condrea. Repeated violations may result in revoked access to the workspace and associated repositories.
